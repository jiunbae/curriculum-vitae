# 면접관 관점 이력서 피드백 (2025-11-22)

## 종합 인상
- 도메인(실시간 음성/3D/디지털 휴먼) 경험은 눈에 띄지만 모든 항목이 결과·규모·품질 지표 없이 “Developed/Improved” 수준에 머물러 영향도를 판단하기 어렵습니다.
- 반복(경력과 프로젝트가 거의 동일한 서술)과 수식어 위주의 표현이 많아 실제 성과보다 역할을 부풀린 인상이 있습니다.
- 기술 스택·배포/운영 역량(가속, 프레임워크, 모니터링, 롤아웃 전략)과 팀 협업 방식이 드러나지 않아 현업 투입 시 리스크가 있습니다.

## 섹션별 비판
- **Return Zero (2024.07–현재)**: “5+ ML-powered features” 등 정량 데이터(배포 유저 수, WER/latency 개선 %, 모델 크기, 비용 절감)와 기술 스택(가속기/서빙 엔진/프롬프트 방식)이 전혀 없음. “Independently owned” 주장 대비 팀 규모, 코드 리뷰/QA/모니터링 체계가 불명확.
- **NCSOFT (2021–2024)**: 3년간 주요 산출물이 “파이프라인 구현/프레임워크 개발/리서치 주도”로만 서술되어 실제 프로덕션 적용 여부, 성능 지표, 배포 범위가 보이지 않음. 프레임워크 생산성 향상 수치나 내부 채택 규모가 빠져 신뢰성이 낮음.
- **프로젝트 섹션**: 경력 섹션과 내용이 대부분 중복되어 길이만 늘어남. 각 프로젝트가 생산한 결과(배포 여부, 사용자/크리에이터 활용 사례, latency·FPS, 품질 지표)가 빠져 있어 임팩트가 판단되지 않음. “Research and experimentation” 같은 모호한 문구가 반복됨.
- **Publications**: 2020년 워크숍/ArXiv 수준 논문만 존재해 최신 전문성 증거가 부족. 저널/컨퍼런스 실적 부재로 연구 깊이에 의문이 생김.
- **학력/기타**: GPA, 논문/프로젝트 링크, 핵심 과목·실습 정보가 없어서 학업 성취도 가늠 불가. TA 항목도 “기본 프로그래밍 지식” 등 모호한 표현뿐이라 교육 효과나 기여도가 드러나지 않음.
- **빠진 정보**: 스킬 섹션 부재로 언어/프레임워크/툴체인/배포 스택을 추정해야 함. 실시간 추론 경험을 주장하지만 어떤 최적화(TensorRT/ONNX/Triton/kv-cache/streaming)와 어떤 하드웨어(T4/A10/Edge GPU/CPU)에서 어느 지연 시간/비용을 달성했는지 알 수 없음. 데이터 책임(수집·어노테이션·윤리/보안)과 운영 책임(SLO, 알람, 롤백 전략)도 드러나지 않음.
- **구조/톤**: 과장된 형용사와 “led/architected/independently owned” 표현이 잦지만 근거 수치가 없고, 최신 수상·오픈소스·특허 등 차별화 요소가 보이지 않아 신뢰도에 타격.

## 추가로 검증하고 싶은 질문
- Return Zero: 현재 서비스되는 모델의 WER/CER, end-to-end latency(차트/분해), 서빙 아키텍처(TRT/Triton/ONNX Runtime/프롬프트 캐싱), 트래픽 규모와 안정성 지표는? 프로덕션 사고 대응 경험은?
- NCSOFT: Digital Human/텍스처 프로젝트가 실제 제품화 또는 데모 단계에서 어떤 KPI를 달성했고, 팀 내 채택률은? 프레임워크가 해결한 구체적 병목(예: 실험 관리/분산 학습/데이터 버전 관리)과 성능 향상 수치는?
- 프로젝트 중복: 경력과 프로젝트를 분리한 기준은 무엇이며, 동일 성과를 중복 기재한 이유는? 한 페이지 내로 임팩트를 압축할 수 있는가?
- 기술 스택: 주력 언어·프레임워크·배포 도구·모니터링 스택, 실시간 최적화 경험(양자화/프루닝/지연 스트리밍/캐시 전략)과 그로 인한 수치적 개선.
- 협업/리더십: 팀 규모, 본인이 설계·코드·리뷰·온콜에서 담당한 범위, 이해관계자(PO/클라이언트/리서쳐)와의 조율 사례.
